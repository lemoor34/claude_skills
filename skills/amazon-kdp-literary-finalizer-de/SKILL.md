## Назначение
Подготовка литературных произведений (романы, повести, серии) к публикации на Amazon KDP Deutschland. Полная проверка качества перед изданием.

## Триггеры использования
- Глава готова, нужна финальная полировка перед публикацией
- Нужна проверка на плагиат и оригинальность
- Требуется форматирование для Amazon KDP (EPUB, KDP format)
- Нужны метаданные (описание книги, ключевые слова, BISAC-категории)
- Требуется анализ читаемости и консистентность психологических линий
- Нужна подготовка обложки и описания для листинга

## Процесс подготовки к Amazon KDP

### Фаза 1: Литературная полировка (Content Quality)

#### 1.1 Проверка прозы
```
Критерии:
✓ Психологическая глубина персонажей
✓ Консистентность голоса автора (не AI-звучит)
✓ Темп повествования (пауза, ускорение, напряжение)
✓ Диалоги звучат естественно
✓ Описания образны, но не перегружены
✓ Метафоры оригинальны и не банальны

Красные флаги:
✗ Повторение одних и тех же фраз (find & replace)
✗ Слишком много наречий (-ly words в английском переводе, -lich в немецком)
✗ Пассивный залог (переделать на активный)
✗ Размытые чувства ("она почувствовала грусть" → "холод сквозь живот")
✗ Exposition на диалогах ("As you know..." — избегать)
✗ Неправдоподобные реакции персонажей
```

#### 1.2 Анализ психологической консистентности
```
Для Romantasy жанра (Die Nebelkrone):

Проверка по каждому персонажу:
- Мотивация (почему он так действует?)
- Развитие (меняется ли за ход сюжета?)
- Страхи и желания (ясны ли они читателю?)
- Реакции (логичны ли в контексте его характера?)
- Диалоговый стиль (у каждого свой или сливаются?)

Примеры вопросов:
- Почему Королева Туманов не просто уничтожит врага одним взмахом?
- Что мешает главной героине сразу понять правду?
- Есть ли у побочных персонажей собственная линия?
```

#### 1.3 Проверка на AI-генерацию
```
Ищем признаки AI-текста (которые НЕ должны быть):
✗ "В этом волшебном мире..." (clichéd openings)
✗ "Она чувствовала смешанные эмоции" (размытость)
✗ Список свойств (вместо показа через действие)
✗ Perfectly balanced sentences (слишком рифмовано)
✗ "The protagonist realized..." структуры
✗ Отсутствие оригинальных метафор

Как звучит хорошо:
✓ Ошибки небольшие (реальных людей опечатки есть)
✓ Неравномерный темп (спешка, потом пауза)
✓ Неожиданные образы (не из учебника)
✓ Противоречия в персонажах (люди сложны)
✓ Живые диалоги (перебивают друг друга, недомолвки)
```

### Фаза 2: Техническое редактирование

#### 2.1 Правописание и грамматика (DE)
```
Инструменты:
- Grammarly (в т.ч. на немецком)
- LanguageTool (бесплатный, немецкий отлично работает)
- Собственное прочтение вслух (ловит неловкость)

Специфика немецкого:
✓ Заглавные буквы в существительных (die Nebelkrone, der König)
✓ Umlauts (ä, ö, ü) — правильное использование
✓ Составные слова (Nebelkrone = Nebel + Krone)
✓ Спряжение глаголов (особенно в прошедшем времени)
✓ Род существительных (der, die, das) — согласование с прилагательными

Частые ошибки в DE:
✗ Англиканизмы (вместо немецких слов)
✗ Неправильное спряжение слабых глаголов
✗ Неправильная пунктуация в придаточных предложениях
✗ Неправильное расположение глагола в предложении
```

#### 2.2 Проверка единообразия (Consistency)
```
Проверяем:
- Названия персонажей (нет ли опечаток)
- Названия мест (Nebelkrone, города, регионы — одинаково везде)
- Магические системы (законы магии не меняются)
- Временная шкала (даты, сезоны — последовательны ли)
- Описания персонажей (цвет волос не меняется без причины)
- Стиль диалогов (каждый персонаж говорит как он)
- Форматирование (курсив, жирный — одинаков везде)

Инструмент: Find & Replace в любом редакторе для проверки вариантов написания
```

#### 2.3 Анализ читаемости
```
Метрики:
- Среднее слово на главу (для Romantasy: 3000-5000 слов)
- Среднее предложение (30-40 слов — хороший баланс)
- Среднее абзац (4-6 предложений)
- Процент диалогов (30-40% от объёма)

Инструменты:
- Flesch-Kincaid Grade (для английского)
- LIX index (для немецкого)
- Собственный подсчёт (ручной анализ темпа)

Цель: текст должен быть приятно читать, не перегружая мозг.
```

### Фаза 3: Форматирование для Amazon KDP

#### 3.1 Подготовка файла DOCX → EPUB/KDP
```
Требования Amazon KDP:

1. Шрифт: Times New Roman или Calibri (стандартные)
2. Размер: 10-12pt
3. Межстрочный интервал: 1.5 или Double (не Single!)
4. Отступ абзаца: 0.5" или 1 см (первая строка)
5. Выравнивание: Justify (по ширине)
6. Без прямых разрывов страниц (Amazon сам разбивает)
7. Структура:
   - Title Page (с названием и автором)
   - Table of Contents (огла��ление)
   - Main Content (главы)
   - About the Author (биография)
   - Back Matter (если есть)

8. Главы: использовать Heading 1 стиль (для TOC)
9. Без специальных символов (только Unicode)
10. Гиперссылки в TOC на главы (для EPUB)
```

#### 3.2 Конвертирование форматов
```
Путь: DOCX → EPUB → KDP

1. Сохраняем как .docx (хороший формат)
2. Конвертируем в EPUB (через Pandoc или Calibre)
3. Проверяем в Kindle Previewer (бесплатный инструмент Amazon)
4. Загружаем в KDP как EPUB или конвертированный в KDP format

Инструменты:
- Calibre (бесплатный, хороший для EPUB)
- Pandoc (технический, для батник-конвертирования)
- Amazon Kindle Previewer (официальный, для проверки)
```

### Фаза 4: Анализ на плагиат

#### 4.1 Проверка оригинальности
```
Ищем:
- Дословные заимствования из известных книг
- Похожие сюжетные ходы (Romantasy архетипы везде, но не должно быть копии)
- Дословные фразы (даже небольшие)

Инструменты:
- Copyscape (платный, но эффективный)
- Turnitin (для текстов в целом)
- Grammarly Plagiarism Checker
- Google Search (скопировать фразу и гуглить)

Для Romantasy важно:
✓ Уникальный world-building (даже если магия похожа)
✓ Оригинальные имена (не копировать из других серий)
✓ Собственные метафоры (не из других авторов)
✓ Уникальный голос (прозу должна быть узнаваемой)

Результат: статья "Оригинальность текста" с % уникальности (должно быть >95%)
```

#### 4.2 Стилометрический анализ (опционально)
```
Проверяем, что текст звучит как один автор (не микс AI и человека):
- Словарь (в среднем сколько уникальных слов)
- Средняя длина слова (в символах)
- Частота используемых слов
- Стилистические особенности

Инструмент: Stylometry анализаторы (продвинутый уровень)
```

### Фаза 5: Метаданные для Amazon KDP

#### 5.1 Название и подзаголовок
```
Правило: Название должно работать в поиске + быть эмоциональным

Пример (Die Nebelkrone):
- Основное: "Die Nebelkrone: Das Erbe der Finsternis"
- Подзаголовок: "Ein Romantasy-Epos über Macht, Liebe und das Schicksal"

Ключевые слова для поиска:
- Romantasy, Magisches Abenteuer, Dunkle Fantasie
- Kindle Unlimited (если в КУ)
- Deutsche Literatur

Оптимизация:
- 1-4 слова основное название (запоминается)
- Подзаголовок описывает жанр и тему
- Ключевые слова — то, что люди ищут
```

#### 5.2 Описание книги (Book Description)
```
Структура (300-400 слов):

1. Hook (2-3 предложения): самая интригующая часть сюжета
   "В королевстве Вечного Туманна правит королева, которая не может умереть.
    Но есть пророчество, что её свергнет девушка, рождённая в грозу."

2. World & Stakes (2-3 абзаца): мир и что на кону
   "В этом мире магия — валюта, и каждое заклинание стоит крови..."

3. Character (1 абзац): главная героиня
   "Элара — сирота с опасным даром, которая не знает свою истинную мощь..."

4. Conflict (1 абзац): основной конфликт
   "Когда королевская стража приходит за ней, Элара вынуждена бежать..."

5. Call to action (2-3 предложения): побуждение купить
   "Следи за судьбой Элары в эпической саге, где магия и любовь встречаются с войной и предательством.
    Если ты любил 'Crown of Thorns' или 'The Poppy War', это для тебя!"

Оптимизация для Amazon:
- Первое предложение должно крючить (это видно в поиске)
- Используй эмодзи экономно (1-2 в начале)
- Ссылки на похожие успешные книги (компарабельные)
- Без очень длинных абзацев (форматирование)
```

#### 5.3 BISAC Categories (Жанровые категории)
```
Для Die Nebelkrone подходят:

Основные:
- FICTION / Fantasy / Dark
- FICTION / Fantasy / Epic
- FICTION / Romance / Fantasy

Дополнительные:
- FICTION / Fantasy / Paranormal
- FICTION / Magical Realism

Совет: выбири 2-3 категории максимум (слишком много = потеря фокуса в поиске)
```

#### 5.4 Ключевые слова (Keywords)
```
Amazon позволяет 7 ключевых слов/фраз (макс 50 символов каждая)

Примеры для Die Nebelkrone:
1. romantasy fantasy deutsch
2. dark fantasy german kindle
3. paranormal romance magic
4. epic adventure novel
5. german fantasy authors
6. paranormal fiction
7. magic and destiny

Правила:
- Выбирай слова, которые люди реально гуглят
- Не набивай одно слово 7 раз
- Используй дефисы и комбинации (romantasy deutsch, не просто romantasy)
- Проверь, нет ли там опечаток
```

#### 5.5 Автор биография
```
Структура (100-200 слов):

"Лена Дунквальд (Lena Dunkwald) — немецко-русская писательница, автор психологической прозы и fantesy-саг.
Её работы исследуют темы идентичности, власти и магии в мирах, где реальность и фантазия переплетаются.

Родилась в России, выросла в Швейцарии, пишет на немецком и русском языках.
Её философский подход к фантастике вдохновлён классической немецкой литературой и современной психологией.

Когда она не пишет, Лена работает в сфере маркетинга и консультирует стартапы.
Она верит, что хорошие истории меняют людей.

Следи за её работами на [website], подписывайся на [threads-ссылка]."

Советы:
- Пиши в третьем лице (даже если это ты)
- Добавь фото (профессиональное, но естественное)
- Дай читателям возможность тебя найти (website, social)
- Будь достоверной (читатели это ценят)
```

### Фаза 6: Обложка и превью

#### 6.1 Требования к обложке Amazon KDP
```
Размеры:
- Минимум: 625 x 1000 px (соотношение 5:8)
- Рекомендуемо: 1500 x 2400 px или выше (для высокого качества)
- Формат: JPG или PNG

Требования:
✓ Шрифт должен быть читаем даже в маленьком размере (обложка в поиске мала)
✓ На корешке должно быть название и автор (для печатных версий)
✓ Нет размытости (300 DPI минимум)
✓ Логотип Amazon не рисуй сам (они добавляют)

Дизайн для Romantasy:
- Тёмные, мистические цвета (фиолет, чёрный, золото)
- Символизм (корона, туман, магические рун)
- Лицо персонажа (если на обложке) — эмоция должна быть ясна
- Шрифт: готический или futuristic (в зависимости от стиля)

Инструменты:
- Canva (простой, есть шаблоны KDP)
- Adobe InDesign (профессиональный)
- Fiverr/Upwork (если нанимаешь дизайнера)
```

#### 6.2 Look Inside (превью в Amazon)
```
Первые 10% книги доступны как превью в Amazon (без платежа).

Оптимизация:
- Первые 2-3 страницы = ДОЛЖНЫ цеплять (это решает покупку)
- Нет грамматических ошибок (первое впечатление)
- Форматирование идеальное (шрифты, отступы, пустые строки)
- Chapter 1 начинается с hook (не с exposition)

Совет: напиши первые 3 абзаца так, чтобы человек НЕ СМОГ остановиться.
```

### Фаза 7: Загрузка на Amazon KDP

#### 7.1 КДП процесс
```
1. Создай аккаунт на kdp.amazon.com (или de.kdp.amazon.com для Германии)
2. Create New Title → Paperback or Ebook
3. Загрузи файл (DOCX или EPUB)
4. Загрузи обложку
5. Заполни все метаданные (BISAC, keywords, description)
6. Выбери цену (минимум 0.99 EUR, максимум — твоё решение)
7. Выбери royalty (35% или 70% — для большинства 70% лучше)
8. Нажми Publish (ждёшь 72 часа)

Совет: сначала опубликуй как Ebook (быстрее и дешевле тестировать).
После успеха → paperback версия (если хочешь)
```

#### 7.2 Kindle Unlimited (КУ)
```
Опция: сделать книгу эксклюзивной для Amazon (180 дней контракт).

Плюсы:
+ Доступ к КУ (читатели платят подписку, деньги распределяют)
+ Boost в алгоритме Amazon
+ Премия за KENPC читаемость

Минусы:
- Не можешь публиковать на других платформах
- Меньше видимости вне Amazon

Совет: для первой книги серии — включи КУ (максимум reach).
Для второй части → можешь расширить на другие платформы (если захочешь).
```

## Чеклист перед публикацией

- [ ] Прочитал вслух (вся книга или хотя бы 5 глав)
- [ ] Грамматика и правописание проверены (LanguageTool + ручное)
- [ ] Консистентность проверена (имена, места, правила магии)
- [ ] Плагиат проверен (>95% оригинальность)
- [ ] Форматирование для KDP правильное (Heading 1 для глав, пробелы верные)
- [ ] Обложка создана (минимум 625x1000px, профессиональный вид)
- [ ] Описание книги написано (300+ слов, с hook)
- [ ] BISAC категории выбраны (2-3 максимум)
- [ ] Ключевые слова подобраны (7 штук, релевантные)
- [ ] Биография автора написана (100-200 слов)
- [ ] Цена выбрана (рекомендую 4.99-6.99 EUR для дебюта)
- [ ] Royalty выбрана (70% лучше всего)
- [ ] Файл загружен в KDP
- [ ] Обложка загружена
- [ ] Все метаданные заполнены правильно
- [ ] Предпросмотр (Look Inside) проверен

## Метрики успеха

**Неделя 1-2:**
- Книга online на Amazon
- Первые читатели нашли (друзья, семья)
- 5-10 первых продаж

**Месяц 1:**
- 20-50 продаж (зависит от маркетинга)
- Первые отзывы (желательно 4-5 звёзд)
- Amazon алгоритм начинает рекомендовать

**Месяц 2-3:**
- Стабильные продажи (если маркетинг идёт)
- 10+ отзывов (социальное доказательство)
- Ranking в категории (лучше — выше в поиске)
